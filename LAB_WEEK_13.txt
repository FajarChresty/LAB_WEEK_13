1. MVVM penting karena pola arsitektur ini memisahkan tampilan (View) dari logika bisnis dan data (Model serta ViewModel), sehingga aplikasi menjadi lebih mudah dikembangkan, di-maintenance, dan diuji. Dengan pemisahan ini, perubahan pada UI tidak akan mempengaruhi logika program, dan perubahan pada logika data tidak akan mengganggu UI. Pada proyek ini, file Movie.kt berperan sebagai Model, file XML seperti activity_main.xml serta MainActivity.kt berperan sebagai View, sedangkan MovieViewModel.kt bertindak sebagai ViewModel yang menghubungkan data dengan tampilan.

2. Data Binding lebih efisien dibandingkan metode biasa karena Data Binding memungkinkan View secara otomatis terhubung dengan data di ViewModel tanpa perlu memanggil findViewById(). Selain itu, UI dapat langsung bereaksi ketika data berubah melalui mekanisme binding yang bersifat reactive. Hal ini mengurangi kode boilerplate, mencegah error referensi ID, dan membuat struktur kode lebih rapi serta mudah dikelola. Dengan Data Binding, RecyclerView dapat langsung menerima data dari ViewModel melalui ekspresi @{}, sehingga tidak perlu lagi menulis observer secara manual.

3. Singleton Pattern penting pada implementasi Room Database karena database adalah objek yang berat untuk dibuat berkali-kali. Dengan menggunakan Singleton, aplikasi hanya membuat satu instance database yang digunakan bersama oleh seluruh komponen. Cara ini mencegah terjadinya race condition, menjaga konsistensi data, serta meningkatkan performa aplikasi. Selain itu, Singleton membantu menghindari konsumsi memori berlebih dan potensi data corrupt akibat multiple instance.

4. Repository Pattern sangat penting karena pola ini menyatukan sumber data dari API dan Room Database dalam satu lapisan yang terstruktur. Dengan adanya repository, ViewModel tidak perlu tahu apakah data diambil dari server atau dari database lokal; cukup memanggil fungsi yang disediakan repository. Repository Pattern juga memungkinkan aplikasi berjalan meski tanpa koneksi internet karena data dapat dibaca dari Room. Selain itu, pola ini membuat kode lebih bersih, modular, mudah diubah, serta lebih mudah diuji.

5. Selain WorkManager, ada beberapa cara lain untuk menyegarkan database dengan data terbaru, seperti melakukan refresh manual menggunakan pull-to-refresh, mengambil data setiap kali aplikasi dibuka atau dilanjutkan (misalnya saat onResume), menggunakan AlarmManager untuk penjadwalan manual, atau menggunakan Foreground Service untuk pembaruan terus-menerus. Namun metode-metode tersebut umumnya kurang efisien, kurang aman, atau lebih boros baterai. WorkManager tetap menjadi cara terbaik karena dirancang khusus untuk menjalankan tugas background secara terjadwal dan tetap berjalan meskipun aplikasi ditutup atau perangkat direstart.
